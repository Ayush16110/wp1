SYNOPSIS

Routines to update the local database from information on the wiki

FUNCTIONS

Standard parameters:
PROJECT

The name of a rated project

EXTRA

The hash ref returned by get_extra_assessments()

download_project_list()

Download list of all participating projects from wiki

Returns array ref

download_project(PROJECT)

Update assessment data for PROJECT

get_project_quality_categories(PROJECT, EXTRA)

Make list of categories storing quality data for project

Each returned rating is a key in %Quality or a value in EXTRA

Returns hash ref: rating => category

get_project_importance_categories(PROJECT, EXTRA)

Make list of categories storing importance data for project

Each returned rating is a key in %Importance or a value in EXTRA

Returns hash ref: rating => category

download_project_assessments(PROJECT, EXTRA, TIMESTAMP, TYPE)

Download assessments for project and update database.

TYPE is either quality or importance.

get_extra_assessments(PROJECT)

Parse the ReleaseVersionParameters template from the main category page for PROJECT

download_review_data()

Download review data from wiki, which concerns FA, GA, etc. Update database.

download_review_data_internal()

Download review data from database. This function does not commit the database.

download_release_data()

Download release data from wiki, which is about release versions such as WP 0.5. Update database.

download_release_data_internal()

Download release data from the database. This function does not commit the database.

update_timestamps( )

Update the internal timestamp variables to the current time.

get_new_name(NS, TITLE, TIMESTAMP)

Tryto get the current location of NS:TITLE, as if it has been moved or has become a redirect. Ignores any moves before TIMESTAMP.

acceptable_namespace(NS)

Used to blacklist undesired namespaces.

Returns 1 if NS is an acceptable namespace for an assessed page, for example namespace 0. Returns 0 for non-acceptable namespaces.
